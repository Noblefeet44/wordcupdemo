⚡ Supabase Setup Guide

The app works with demo data by default. Follow these steps to connect your own Supabase database.

1. Create a Supabase project

Go to supabase.com, create a free account and a new project.

2. Run this SQL in your Supabase SQL editor

-- Events table
CREATE TABLE events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  team1 TEXT NOT NULL,
  team2 TEXT NOT NULL,
  match_type TEXT,
  match_date DATE,
  match_time TIME,
  country TEXT,
  city TEXT,
  stadium TEXT,
  badges TEXT[],
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Ticket categories table
CREATE TABLE ticket_categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  event_id UUID REFERENCES events(id) ON DELETE CASCADE,
  category_name TEXT,
  price NUMERIC,
  currency TEXT DEFAULT 'USD',
  is_available BOOLEAN DEFAULT true
);

-- Enable Row Level Security (public read)
ALTER TABLE events ENABLE ROW LEVEL SECURITY;
ALTER TABLE ticket_categories ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read events" ON events FOR SELECT USING (true);
CREATE POLICY "Public read categories" ON ticket_categories FOR SELECT USING (true);

-- Service role full access (for admin)
CREATE POLICY "Service role all events" ON events FOR ALL USING (auth.role() = 'service_role');
CREATE POLICY "Service role all categories" ON ticket_categories FOR ALL USING (auth.role() = 'service_role');

3. Create a .env file in the project root

VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here

Find these in your Supabase project → Settings → API.

4. Restart the dev server

npm run dev

The demo banner will disappear and you can start adding real events from this dashboard.
